## Did the changes land as anticipated?

Mostly yes.

- The proof-complete policy eliminated the three previous false passes.
- Every recorded run used claim-by-claim verification; the permissive single-verdict fallback was not used.
- Snapshot settling and action recovery increased result-page completion from 2/20 to 18/20.
- No run used browser back.
- Named-group enrichment only landed partially. The calculator does not consistently expose semantic labels for Workday, Schedule, Weekend, and similar controls.
- Settling still sometimes accepts an intermediate transition. The new traces contain 33 stale-reference and 34 hidden-target errors. Most recovered, but they remain technical friction.
- Proof-complete verification traded false positives for false negatives. The main remaining work is group evidence, conditional claim decomposition, and final-summary verification.
- Run 20 is a separate artifact-writing failure and should be investigated independently.

## Overall conclusion

Terminal technical reliability improved substantially and false passes were eliminated. The unchanged reported pass rate is now dominated by verifier and test-observability problems rather than failure to complete the calculator.

This is strong evidence that the changes worked for this calculator flow, but it is not yet evidence of equivalent improvement across websites. Later test reports should retain the same core metrics so the aggregate picture can distinguish browser execution, verifier accuracy, product/test observability, and result-recording faults.
